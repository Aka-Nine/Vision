Major Subsystems

🌐 Ingestion Layer

Connects to external APIs (Reddit PRAW, HackerNews, PyTrends) to identify statistically relevant design trends. Includes caching and rate-limiting to protect external quota.

🎭 Scraping Layer

Uses Playwright headless automation to physically navigate websites, authenticate into Pinterest, and download high-res UI references. User-agent rotation prevents blocks.

🧠 Analysis Engine

Multi-modal visual inference using Gemini 2.5 Flash. Translates raw pixels into JSON layout structures with spatial bounding box coordinates [x,y,w,h] for every component.

⚖️ Evaluation Engine

Deterministic scoring pipeline: Reputation Scorer → Deduplication → Opportunity Ranker → Brief Scorer → Generator Validator. Ensures only high-quality, viable briefs reach Phase 3.

🗃️ Storage Layer

Out-of-core persistence: flat-file JSON database (market_data.json), image index, pattern library files, pipeline state tracker, and automated dataset lifecycle management.

📡 Orchestration Layer

Event-driven pipeline engine with exponential backoff retries, Prometheus metrics, secret-masking logger, and a scheduler for autonomous 12-hour market scan cycles.

How MCP Brain Wraps Phase 2

The MarketPipeline class wraps the entire agent execution in an event-driven retry loop. It emits events to the central EventBus at start/complete/fail, updates the StateTracker at each stage, and increments Prometheus Metrics for observability. On failure, asyncio.sleep(2^attempt) provides exponential backoff before retry.

Pipeline Walkthrough

Simulating a complete execution for topic: "AI SaaS dashboard". Click each step to expand.

01
Trend Collection
trend_collector.py · cache.py

The cache is checked first. If miss, PRAW connects to Reddit. With a 401 error (no valid creds), it falls back to HackerNews REST. The top story "AI Code Tools Are Replacing Devs" is normalized to a Trend Object. The result is written to cache/latest_trend.json with a TTL of 3600 seconds.

trend: "AI Code Tools UI" score: 0.88 source: hacker_news_free_api
02
Rate Limiter + Design Scraping
rate_limiter.py · design_scraper.py

Rate limiter waits 2.7 seconds (base 2s + jitter 0.7s) and selects a random user-agent. Pinterest credentials are detected in .env — Playwright launches headless=False, logs in, searches "AI Code Tools UI design" and extracts 4 high-res 736x PNG image URLs. Results are cached by topic key.

image_urls: [4 Pinterest URLs] source: pinterest reputation: 0.6
03
Screenshot Capture
design_scraper.scrape_live_website()

If the trend contained a reference_url (from a Reddit post), Playwright navigates to the live site, scrolls 1000px to trigger lazy-loaded elements, waits 2 seconds for animations to settle, and takes a full-page screenshot saved to storage/visuals/live_scrape_HASH.png.

local_screenshot: storage/visuals/live_scrape_X.png
04
Visual Analysis (Gemini 2.5 Flash)
trend_analyzer.py

The 4 Pinterest image URLs are fetched (httpx on-the-fly), raw bytes are bundled with the text prompt into a multi-part Gemini payload. The model physically sees the pixels and returns a JSON structure identifying sections, typography, colors, animations, and bounding box coordinates for each UI block.

sections: ["sidebar","metrics_grid","chart"] annotations: [3 coord objects]
05
Pattern Extraction + Library Save
pattern_extractor.py · structured_pattern_library.py

The analyzer output is normalized into a canonical UI Pattern Object. Simultaneously, pattern_library.save_pattern('glassmorphic_dark', pattern) appends the pattern to patterns/glassmorphic_dark_patterns.json, growing the reusable library for future runs.

layout_type: dashboard style: glassmorphic_dark patterns/glassmorphic_dark_patterns.json updated
06
Demand Estimation
demand_estimator.py · PyTrends

The keyword "dashboard template" is sent to PyTrends querying Google Trends for the past 30 days. Average normalized interest returns 0.79. Competition is hardcoded at 0.45 (extensible via SerpAPI later).

demand_score: 0.79 competition_score: 0.45
07
Opportunity Ranking
opportunity_ranker.py · reputation_scorer.py

Source reputation (Pinterest = 0.6) is attached to the designs dict. Opportunity score = 0.79 × 1.0 × (1 - 0.45) = 0.435. If multiple demand objects exist, the highest-scoring one is selected as best_opp for brief generation.

opportunity_score: 0.435 reputation: 0.6
08
Design Brief Generation (Dual-LLM: LLaMA → Gemini)
design_brief_generator.py · core/llm_gateway.py

All gathered intelligence is sent through the Dual-LLM Gateway. Step 1: LLaMA 3.2 (local, via Ollama) produces a fast JSON draft (~13s, zero API cost). Step 2: Gemini 2.5 Pro refines and enhances the draft (when quota is available). If Gemini is rate-limited, the LLaMA draft is used directly. Markdown fences are stripped, demand_score is re-stamped.

product_type: "AI SaaS Dashboard Template" provider: dual (llama+gemini)
09
Validation Cascade
brief_scorer → generator_validator → brief_validator

1) BriefScorer computes quality_score = 0.87 (style ✓ 0.3, sections ✓ 0.3, animation ✓ 0.2, demand ✓ 0.158). 2) GeneratorValidator checks sections non-empty, style truthy, quality ≥ 0.6 — all pass. 3) BriefValidator confirms all required keys populated. Provenance dict is injected.

quality_score: 0.87 compatible: True valid: True
10
Storage & Cleanup
metadata_store · state_tracker · dataset_manager

The validated brief is appended to market_data.json. StateTracker marks pipeline "completed" with end_time. DatasetManager scans images folder — 12 files found, none older than 30 days, no archiving needed. Metrics counters (brief_generation_total, pipeline_success_total) are incremented. Pipeline returns successfully.

market_data.json updated status: completed

Component Interaction Map

How modules communicate through data passing, events, and side effects.

Interaction Sequence

  1. Pipeline Engine → Agent: MarketPipeline.run() calls MarketIntelligenceAgent.process() inside a retry while-loop. StateTracker.create_pipeline() is called before any agent work begins.
  2. Agent → EventBus: Every major event fires log_activity() which wraps event_bus.emit(). The Bus broadcasts to all subscribers (currently logger + monitor hooks).
  3. TrendCollector → Cache: Before any API call, cache.get() is checked. On hit, the cached dict is returned directly. On miss, API is called and cache.set() saves the result with TTL=3600.
  4. Scraper → RateLimiter: Every Playwright launch is gated through rate_limiter.wait() which enforces randomized jitter delays and injects a rotated User-Agent string into context headers.
  5. Agent → DeduplicationEngine: After scraping, deduplication_engine.is_duplicate(designs) is called. Skip metrics increment on True. This is a side-channel check — not blocking pipeline flow.
  6. PatternExtractor → PatternLibrary: After normalization, pattern_library.save_pattern() is called as a side effect, persisting to disk without interrupting the main data flow.
  7. BriefGenerator → Validation Cascade: brief_scorer.score_brief() → generator_validator.is_compatible() → brief_validator.validate() run sequentially, each throwing an Exception on failure which bubbles up to the pipeline retry loop.
  8. Pipeline → StateTracker: On success: state_tracker.complete_pipeline(). On failure: state_tracker.fail_pipeline(error). Either way, the run is permanently logged for post-mortem analysis.

Failure Scenarios

Click each scenario to see the full recovery path.

🟠 Reddit API Returns 401 Unauthorized
Authentication failure · PRAW

Root Cause: Reddit credentials not set in .env, or app token expired.

Recovery Path: The except Exception block in trend_collector.py catches the error and logs it via logging.error("Reddit API failed: ..."). Execution falls through to the HackerNews REST API path. If HN also fails, the deterministic fallback dict {"trend": "AI SaaS landing page", "score": 0.89} is returned. The pipeline never receives None.

Prevention: Set REDDIT_CLIENT_ID and REDDIT_CLIENT_SECRET in .env. The validate_environment() call on startup will warn immediately if missing.

🔴 Playwright Scraping Times Out / CAPTCHA Block
Browser automation failure · design_scraper.py

Root Cause: Pinterest or Dribbble detected bot activity, served CAPTCHA, or selector timeout hit 15 seconds.

Recovery Path: The try/except Exception in each scrape method catches all Playwright errors. The fallback sets image_urls: ["https://placehold.co/800x600/png?text=Blocked"] — a real URL that doesn't crash downstream byte fetching. The rate limiter's jitter and UA rotation reduce future occurrence probability.

Prevention: Increase jitter range. Use rotating residential proxies. Consider Apify cloud scraper actors which are maintained to bypass modern bot detection.

🟣 Gemini API Returns 429 Quota Exceeded
LLM rate limit · trend_analyzer.py + design_brief_generator.py

Root Cause: Free-tier Gemini API quota consumed (typically 15 RPM / 1M TPD for Flash, lower for Pro).

Recovery Path [Analyzer]: Returns {"popular_sections": ["hero with gradient", "feature grid", "pricing table"], "error": "..."} — a minimal but valid response that allows pattern extraction to proceed.

Recovery Path [Generator]: Falls back to an algorithmically composed brief using the collected demand, pattern, and trend data directly — populating all required fields without LLM involvement. The brief_validator then catches it, and in most cases it passes.

Prevention: Implement request queuing with exponential backoff (already in pipeline retry loop). Upgrade to paid Gemini API tier. Add Claude or GPT-4o as secondary LLM fallbackswhen primary quota exceeds.

🟡 Invalid Brief Generated — Validation Fails
Schema violation · generator_validator.py + brief_validator.py

Root Cause: LLM generated a brief missing required keys (e.g., product_type absent), or quality_score computed below 0.6 threshold (too few sections, no style defined).

Recovery Path: generator_validator.is_compatible() returns (False, reason). The agent calls log_activity("generator_compatibility_failed") then raise Exception(f"Unstable Brief Generated: {reason}"). This propagates up to MarketPipeline's retry while-loop. Attempt counter increments, state_tracker records the error string, asyncio.sleep(2^attempt) fires, then the entire pipeline stage restarts.

Max Retries: 3 attempts (configurable via self.max_retries). After 3 failures, state_tracker.fail_pipeline() is called and the exception re-raises to the caller.

🟢 Duplicate Design Detected by Deduplication Engine
Soft failure · deduplication_engine.py

Root Cause: The same trend topic was scraped multiple times (e.g., cache expired, same HN story trending across days). The MD5 hash of the designs dict matches a previously seen hash.

Recovery Path: This is a "soft" failure — the pipeline does NOT stop. Instead, metrics.scrape_success.inc() is skipped (keeping metrics accurate), and log_activity("duplicate_designs_skipped") is emitted. The pipeline continues to analysis and brief generation — sometimes producing a differently ranked brief even from duplicate source data due to changing demand scores.

Improvement: Current in-memory hash set resets on restart. Persisting hashes to Redis with 7-day TTL would enable true cross-run deduplication.

Engineer Onboarding Guide

Welcome to the team. Here is everything you need to understand this system.

What Problem Does This System Solve?

Before Phase 2 existed, our Template Factory (Phase 3) needed a human to manually specify what to build: "build a dark SaaS dashboard." That is not scalable. We want a system that autonomously discovers what the market wants, analyzes what top designers are building, and continuously feeds structured specifications into the generation pipeline — completely hands-off.

Phase 2 is that system. It is the autonomous market research arm of the product factory.

The Mental Model

📡 Antenna

TrendCollector + Scheduler continuously monitors Reddit/HN for market signals — what topics are designers discussing right now?

👀 Eyes

DesignScraper + TrendAnalyzer physically browse the internet, download images, and use Gemini Vision to understand what they see at the pixel level.

🧠 Brain

PatternExtractor + DemandEstimator + OpportunityRanker analyzes viability — is this trend actually worth building? What's the demand score?

📋 Mouth

DesignBriefGenerator + Validators produce the final standardized specification — a JSON "design ticket" that Phase 3 ingests to generate code.

What Is the Final Output?

A Design Brief JSON object stored in market_data.json. Think of it as a Jira ticket written by an AI Product Manager, specifically formatted for a robot programmer. It contains the product type, target customer, visual style, ordered UI sections, animation type, a quality score, and full provenance data tracing how every field was derived.

Phase 3 reads this brief to generate actual React/Tailwind/HTML files. The richer and more accurate the brief, the higher quality the generated template output will be.

Future Growth Directions

🚀 Scale (6 months)

Redis caching + Celery distributed workers + PostgreSQL storage. Run 50 parallel pipeline scans instead of 1 sequential scan.

🤖 Intelligence (1 year)
